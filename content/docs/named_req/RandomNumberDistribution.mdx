# <Since v='cpp11'><small>C++                    named requirements:</small><br/>*RandomNumberDistribution*</Since>

A ***RandomNumberDistribution*** is a function object returning random numbers according to a 
[probability density function](https://en.wikipedia.org/wiki/Probability_density_function) *p(x)* or a 
[discrete probability distribution](https://en.wikipedia.org/wiki/Probability_distribution#Discrete_probability_distribution) *P(x<sub>i</sub>)*.

### Requirements
The type `D` satisfies ***RandomNumberDistribution*** if

* **D** satisfies [*CopyConstructible*](/docs/named_req/CopyConstructible)
* **D** satisfies [*CopyAssignable*](/docs/named_req/CopyAssignable) and

Given

* `T`, the type                  named by `D::result_type`
* `P`, the type                  named by `D::param_type`, which
    * satisfies [*CopyConstructible*](/docs/named_req/CopyConstructible)
    * satisfies [*CopyAssignable*](/docs/named_req/CopyAssignable)
    * satisfies [*EqualityComparable*](/docs/named_req/EqualityComparable)
    * has a constructor taking identical arguments as each of the constructors of D that take arguments corresponding to the distribution parameters.
    * has a member function with the identical                   name, type, and semantics, as every member function of D that returns a parameter of the distribution
    * declares a member typedef `using distribution_type = D;`
* `d`, a value of type **D**
* `x` and `y`, (possibly const) values of type **D**
* `p`, a (possibly const) value of type **P**
* `g`, `g1`, `g2`, lvalues of a type satisfying [*UniformRandomBitGenerator*](/docs/named_req/UniformRandomBitGenerator)
* `os`, lvalue of a specialization of [std::basic_ostream](/docs/std/io/basic_ostream)
* `is`, lvalue of a specialization of [std::basic_istream](/docs/std/io/basic_istream)

The following expressions must be valid and have their specified effects

| `x.min()`        | **T**                         | the greatest lower bound on the values potentially returned by `x`’s operator(), as determined by the current values of `x`’s parameters                                                                                                                                                                                                                                                                                                                              | constant                                        |

| `x != y`         | `bool`                        | `!(x == y)`                                                                                                                                                                                                                                                                                                                                                                                                                                                           | constant                                        |

The formatting flags and fill character of `os` are unchanged.                                                                                                                                                                                                                                                                                                             |                                                 |
| `is >> d`        | Reference to the type of `is` | Restores the distribution parameters and internal state with data read from `is`. The formatting flags of `is` are unchanged. If bad input is encountered, `is.setstate(std::ios::failbit)` is called, which may throw [std::ios_base::failure](/docs/std/io/ios_base/failure). `d` is unchanged in that case. |                                                 |

### Notes
The parameters of a distribution object may be changed either permanently, by using `d.param(p)` or just for the duration of a single operator() call, 
by using `d(g,p)`.

Calls to const member functions of the distribution and `os << d` do not affect the sequence of numbers produced by repeated `d(g)`.
