# Replacing text macros
The preprocessor supports text macro replacement. Function-like text macro replacement is also supported.

## Syntax

<SymbolTable noTraits>
  <Symbol name={<><code>1</code> <code-d>#define</code-d></>} desc={<><i>identifier replacement-list</i><small class="col-d"> (optional)</small></>}/>
  <Symbol name={<><code>2</code> <code-d>#define</code-d></>} desc={<><i>identifier (parameters) replacement-list</i><small class="col-d"> (optional)</small></>}/>
  <Symbol name={<><code>3</code> <code-d>#define</code-d></>} desc={<><i>identifier (parameters, ...) replacement-list</i><small class="col-d"> (optional)</small><Since v='cpp11'></Since></>}/>
  <Symbol name={<><code>4</code> <code-d>#define</code-d></>} desc={<><i>identifier (...) replacement-list</i><small class="col-d"> (optional)</small><Since v='cpp11'></Since></>}/>
  <Symbol name={<><code>5</code> <code-d>#undef</code-d></>}  desc={<i>identifier</i>}/>
</SymbolTable>

## Explanation
### <span class="col-d">#define</span> directives
The #define directives define the identifier as macro, that is instruct the compiler to replace most successive occurrences of identifier with replacement-list, which will be additionally processed. Exceptions arise from the rules of scanning and replacement. If the identifier is already defined as any type of macro, the program is ill-formed unless the definitions are identical.
