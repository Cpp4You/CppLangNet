# Zmienne

W tej lekcji powiemy o tym, jak przechowywać różne wartości w trakcie
działania programu, czyli jak korzystać ze zmiennych.

## Motywacja

Żeby zrozumieć do czego są one przydatne, napiszmy program,
który podnosi pewną liczbę do potęgi trzeciej, a następnie ją wyświetla:

<Columns spacing={40} columns={['8fr', '4fr']}>
<div>

```cpp title="🔹 main.cpp | Potęga trzecia"
#include <iostream>

int main()
{
	std::cout << 212 * 212 * 212;
}
```

</div>

<div>

```plaintext title="Wynik"
9528128
```

</div>
</Columns>

Jeśli teraz będziemy chcieli podnieść do trzeciej potęgi inną liczbę, będziemy
musieli zmieniać trzy miejsca w kodzie. Jest to uciążliwe.

## Wprowadzenie

Żeby nie zmieniać za każdym razem tylu rzeczy, załóżmy na chwilę, że liczba, którą podnosimy
do potęgi nazywa się `x`. Jak wtedy będzie wyglądał kod?

```cpp
#include <iostream>
int main()
{
	std::cout << x * x * x;
}
```

Powyższy program na tym etapie, **nie skompiluje się**. Żeby użyć `x` w obliczeniu, musimy napierw
stworzyć **zmienną** właśnie o nazwie `x`:

```cpp
#include <iostream>
int main()
{
	int x = 212;
	std::cout << x * x * x;
}
```

Powyższy program skompiluje się i zadziała poprawnie.

:::important Zmienna
**Zmienna** to pojemnik na dane **określonego typu**.
:::

### Typ zmiennej

Typ zmiennej mówi o tym, co może dana zmienna przechowywać.
Tak jak wspominałem wcześniej, zmienna jest pojemnikiem przechowującym
pewne rzeczy, **określonego typu**.

`int` to skrót od angielskiego słowa *integer*, który oznacza **liczbę całkowitą**,
czyli taką bez części ułamkowej.

Na tym etapie powinieneś zapamiętać następujące typy zmiennych:

Typ | Opis
--- | ---
**`int`** | liczba całkowita
**`float`** | liczba rzeczywista (z częścią ułamkową)
**`char`** | pojedynczy znak
**`bool`** | wartość logiczna, czyli prawda (**`true`**) albo fałsz (**`false`**)

:::danger Uwaga
Wyżej wymienione typy zmiennych zawsze piszemy **małą literą**. Ma to znaczenie.
:::

### Nazwa zmiennej

Nazwa może być dowolna, z pewnymi istotnymi ograniczeniami:

- ❌ nie może posiadać spacji (zabronione: ❌ `abc def`)
- ✔ może składać się z:
	- ✔ znaków `a`-`z` oraz `A`-`Z`
	- ✔ cyfr, ale nie na początku (zabronione: ❌ `932abc`)
	- ✔ podkreślnika `_`

Możesz utworzyć tyle zmiennych ile tylko chcesz, jednak muszą one różnić się nazwami,
bo są ich unikalnymi identyfikatorami (z pewnymi wyjątkami, ale o tym kiedy indziej).

Wielkość liter ma znaczenie:

```cpp
int x = 30;
float X = 12.34f;
```

W wyżej przedstawionym kodzie, zmienne `x` oraz `X` to dwie różne zmienne!

:::tip Dobra praktyka
Najlepiej korzystaj z **tylko** angielskich nazw, czyli np:
```cpp
float average;
```

zamiast

```cpp
float srednia;
```
:::

### Inicjalizacja (wartość początkowa)

Inicjalizacja jest **opcjonalnym** krokiem w tworzeniu zmiennych.
Gdy ją kreujemy, możemy albo zostawić ją bez chcianej wartości:

```cpp
int x;
```

lub przypisać jej wartość początkową:

```cpp
int x = 30;
```

Uwaga, słowo to: "inicjalizacja" ✔, nie "inicjacja" ❌

:::caution Brak inicjalizacji
Niezainicjalizowana zmienna, wewnątrz bloku funkcji (np. wewnątrz `main`-a)
będzie na początku posiadała przypadkową wartość, **nie będzie wyzerowana**.

Zostawienie zmiennej bez wartości początkowej **może powodować problemy**
w dalszej części kodu. Czasami jednak intencjonalnie możemy to pomijać,
ale o tym później.
:::


### Średnik na końcu

Tylko dla przypomnienia 😀

```cpp
int name = 40;
```

## Możliwości

Zmienne przechowują wartości przez czas swojego istnienia (o tym później).
Możemy do nich zapisywać wartości lub je odczytywać, wyświetlać na ekranie,
porównywać itp.

### Prosta interakcja z użytkownikiem

Nasze programy będą nieco ciekawsze, gdy pozwolimy użytkownikowi
wpisać coś do konsoli.  
Zrobimy to za pomocą instrukcji `cin` (ang: <i>**c**haracter **in**put</i>):

```cpp
#include <iostream>

int main()
{
	std::cout << "Podaj swój wiek: ";
	
	int age;
	// highlight-next-line
	std::cin >> age;

	std::cout << "Masz " << age << "lat.\n";
}
```

`cin` pozwala nam zapisać zawartość tzw. **standardowego wejścia** (stdin)
do zmiennej, którą podajemy po znakach `>>`.

O standardowym wejściu powiemy kiedy indziej, na razie zapamiętaj, że
`cin` umożliwia użytkownikowi programu wpisać coś od konsoli,
po czym zapisuje ową wartość do zmiennej.

Zwróć uwagę, że:
- `cout` używa operatora `<<`
- `cin` używa operatora `>>`

Oprócz tego, możesz zobaczyć, że **nie przypisaliśmy zmiennej `age` wartości początkowej.**
Nie jest to potrzebne, bo od razu po jej stworzeniu, używamy `std::cin` aby zapisać
do niej wartość taką, jaką użytkownik wpisze z klawiatury.

:::tip Zapamiętaj
Możesz zapamiętać kierunek znaczków w tym operatorze w prosty sposób
- `cout` - tekst idzie **do konsoli** - `<<` skierowane w strone `cout`
- `cin` - tekst idzie **do zmiennej** - `>>` skierowane w stronę zmiennej
:::


### Proste operacje matematyczne

Zmienne, które zawierają liczby można dowolnie modyfikować
za pomocą typowych znaków (operatorów) matematycznych:

- `a + b` - dodaje do siebie dwie liczby
- `a - b` - odejmuje od siebie dwie liczby
- `a * b` - mnoży przez siebie dwie liczby
- `a / b` - dzieli liczbę `a` przez `b`

:::note Uwaga
To nie są wszystkie operatory matematyczne, o reszcie powiemy
kiedy indziej.
:::

Wykorzystajmy te operatory w praktyce:

```cpp
#include <iostream>

int main()
{
	std::cout << "Podaj swój wiek: ";

	int age;
	std::cin >> age;

	// highlight-start
	std::cout << "Za 10 lat, będziesz mieć: "					<< (age + 10)	<< " lat\n";
	std::cout << "5 lat temu, miałeś/aś: "						<< (age - 5)	<< " lat\n";
	std::cout << "Jak będziesz 2x starszy/a, będziesz mieć: "	<< (age * 2)	<< " lat\n";
	std::cout << "Ktoś dwa razy młodszy ma :"					<< (age / 2)	<< " lat";
	// highlight-end
}
```

:::tip Wyrównanie
Zwróć uwagę, że dla lepszej czytelności kodu wyrównaliśmy klawiszem **`Tab`**
użycia operatora `<<`.
:::

Bardzo istotnym faktem jest to, że użycie wyżej wymienionych operatorów,
**nie zmienia zawartości** zmiennej `age`.

Wpisując wiek `20`, uzyskamy następujący wynik:

Jak widzimy, na każdym etapie wartość zmiennej `age` pozostała ta sama (`20`).
Dzieje się tak dlatego, że w zapisie:
```cpp
a + b
```
to samo **wyrażenie** `a + b` ma pewną wartość, a zawartość zmiennych zostaje nietknięta.

Jeśli chcemy np. zwiększyć zmienną o `10`, to musimy **przypisać** do niej
**wartość** tego wyrażenia:

```cpp
age = age + 10;
```

... lub w skrócie:

```cpp
age += 10;
```

- podczas wykonania linii `11` ma wartość `20`.
